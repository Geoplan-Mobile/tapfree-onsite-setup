# 02. LTE 모뎀 설정

← [01. 하드웨어 배선](./01-hardware-wiring.md)

EdgePC 가 외부망에 접속하기 위한 LTE 모뎀 기본 설정 단계.
모바일 클라이언트는 이 모뎀의 외부 IP / DDNS 로 EdgePC 의 Tapfree-edge 서버에 접속한다.


> EdgePC 자체에는 별도의 디스플레이/입력 장치가 상시 연결되어 있지 않으며,
> <br> 원격 접근(SSH)은 **LTE 모뎀이 만들어 주는 네트워크에 합류한 단말에서만** 가능하다.<br>
> 즉, LTE 모뎀이 정상 설정되어 네트워크가 살아 있어야 노트북/스마트폰에서 EdgePC 로 접근할 수 있다.
>
> LTE 모뎀 설정 이전 단계에서 EdgePC 에 접근해야 하는 경우에는,<br>
> **EdgePC 에 모니터와 키보드를 직접 연결**하여 로컬 콘솔로 작업해야 한다.

---

## 사용 모뎀

**표준 채택 모델은 없다.** 제조사 / 모델은 자유롭게 선택할 수 있으나, 아래 기능을 모두 설정할 수 있는 모델이어야 한다.

각 기능은 본 문서의 이후 절에서 하나씩 다룬다.

| # | 설정 항목 |
|---|---|
| 1 | 내부 네트워크 설정 (모뎀 내부 IP 포함) |
| 2 | DHCP 설정 |
| 3 | DDNS 설정 |
| 4 | DMZ 설정 |
| 5 | 모뎀 설정 페이지 외부 Port 설정 |
| 6 | Wi-Fi 설정 |

> 메뉴 경로 / 입력 양식은 모뎀 모델마다 다르므로, 본 문서는 **설정해야 할 값과 그 의미** 중심으로 기술한다. 실제 메뉴 위치는 모뎀 매뉴얼을 참조한다.

---

## 1. 내부 네트워크 설정

모뎀이 만들어 주는 내부 LAN 의 IP 대역과 모뎀 자신의 내부 IP 를 지정한다.

| 항목 | 값 |
|---|---|
| 모뎀 내부 IP | `192.168.0.29` |
| 서브넷 마스크 | `255.255.255.0` |

여기서 지정한 **모뎀 내부 IP 는 곧 모뎀 설정 페이지의 접속 IP** 가 된다.

- 설정 페이지의 **포트와 경로(주소)는 제조사마다 다르므로 확인하여 메모**해 둔다. (예: LG 모뎀은 `80` 포트, `path` 없음)
- 설정 페이지 접속용 **ID / 비밀번호도 함께 메모**해 둔다.

본 설정 이후부터는 내부 네트워크에 연결된 기기에서 **`http://192.168.0.29:{port}/{path}`** 로 모뎀 설정 페이지에 접속할 수 있다.

> 성남 캠퍼스의 경우 내부 네트워크 상태에서 **`http://192.168.0.29:80`** 으로 접속 가능 (LG 모뎀, `path` 없음).

---

## 2. DHCP 설정

내부 네트워크에 합류하는 기기에 자동으로 IP 를 할당하기 위한 DHCP 풀 대역을 설정한다.

| 항목 | 값 |
|---|---|
| DHCP 할당 시작 주소 | `192.168.0.31` |

`192.168.0.29` 와 `192.168.0.30` 은 각각 모뎀과 EdgePC 가 **고정 IP(static)** 로 사용하므로 DHCP 풀에 포함되지 않도록 한다.

| IP | 용도 | 할당 방식 |
|---|---|---|
| `192.168.0.29` | 모뎀 자신 | static |
| `192.168.0.30` | EdgePC | static |
| `192.168.0.31` 이후 | 기타 내부 기기 (Quuppa 앵커, 모니터링 device 등) | DHCP |

---

## 3. DDNS 설정

외부에서 도메인으로 본 사이트에 접속할 수 있도록 DDNS 호스트명을 등록한다.

### 호스트명 규칙

```
tapfree{XXX}{YY}.{ddns_domain}
```

| 부분 | 의미 | 비고 |
|---|---|---|
| `XXX` | **Station ID** | 게이트가 Edge Server 에 접속할 때 전달하는 `stationId` 값과 **반드시 일치**해야 한다. |
| `YY` | **Zone ID** | 현재 1개 Zone 기준이므로 **`01`** 로 고정한다. (추후 Edge Server 설정과 일치시켜야 함) |
| `{ddns_domain}` | DDNS 도메인 | 모뎀 자체 DDNS 또는 외부 DDNS 서비스 등록을 통해 결정된다. (아래 참조) |

### `ddns_domain` 결정

`ddns_domain` 은 다음 중 한 가지 방법으로 결정된다.

- **모뎀 자체에서 DDNS 도메인을 제공**하는 경우 — 모뎀이 제공하는 도메인을 그대로 사용.
- **외부 DDNS 서비스에 등록**하는 경우 — 등록한 서비스의 도메인을 사용.

어느 방법이든 `ddns_domain` 값은 **임의로 변경할 수 있는 값이 아니며**, 결정된 후에는 그대로 사용해야 한다.

> **결정된 `ddns_domain` 값은 반드시 메모해 둔다.**
> 이 값은 추후 모바일 앱 구동 시 SDK 에 전달되어야 한다.

### 예시 — 성남 캠퍼스

성남 캠퍼스는 **LGU+ 모뎀이 제공하는 DDNS 도메인 `cns-link.net`** 을 사용한다.

| 항목 | 값 |
|---|---|
| Station ID | `E08` |
| Zone ID | `01` |
| `ddns_domain` | `cns-link.net` (LGU+ 모뎀 제공) |
| 호스트명 | `tapfreeE0801.cns-link.net` |

---

## 4. DMZ 설정

외부 네트워크(DDNS) 를 통해 들어오는 **모든 요청을 내부 네트워크의 특정 IP 로 그대로 전달**하기 위해 DMZ 를 설정한다.

| 항목 | 값 |
|---|---|
| DMZ 대상 IP | `192.168.0.30` (EdgePC) |

> 본 설정 이후, 외부에서 `https://tapfreeE0801.cns-link.net:9443/` 로 들어오는 요청은 모뎀이 EdgePC(`192.168.0.30`) 로 그대로 전달한다.

---

## 5. 모뎀 설정 페이지 외부 Port 설정

DMZ 설정에 의해 **모든 포트의 외부 요청이 EdgePC(`192.168.0.30`) 로 전달**되므로, 외부에서 모뎀 설정 페이지에 접속할 방법이 없어진다.

이를 해결하기 위해 모뎀 설정 페이지에 **별도의 외부 포트**를 지정하여 DMZ 우회 경로를 만든다.

| 항목 | 값 |
|---|---|
| 외부 접속용 포트 | `8081` |

> 본 사이트 표준값은 `8081` 이다. (LGU+ 모뎀 작업 시 `8081` 로 지정하여 사용한 사례에 따른 표준화)

본 설정 이후 외부에서 다음 주소로 모뎀 설정 페이지에 접속할 수 있다.

```
http://tapfreeE0801.cns-link.net:8081/{path}
```

> `{path}` 는 모뎀 모델에 따라 다르며, [1. 내부 네트워크 설정](#1-내부-네트워크-설정) 단계에서 메모해 둔 경로를 사용한다.
>
> **LGU+ 모뎀**의 경우 `8081` 포트에 별도 `{path}` 가 없다. 즉, `http://tapfreeE0801.cns-link.net:8081` 로 바로 접속 가능.

---

## 6. Wi-Fi 설정

본 설정은 **프로젝트의 필수 설정은 아니다.**  
DDNS 또는 외부 접근에 문제가 생겼을 때, 내부 네트워크를 통해 EdgePC 등 내부 장비에 접근하여 점검하기 위한 **편의용 설정**이다.

- DDNS / 외부 접속이 정상일 때는 사용할 필요가 없다.
- DDNS / 외부 접속이 불가능한 경우, 노트북/스마트폰을 본 Wi-Fi 에 연결한 뒤 <br>EdgePC 의 내부 IP(예: `192.168.0.29`, `192.168.0.30`) 로 접속하여 설정을 점검할 수 있다.
- Wi-Fi 를 사용하지 않고 **LTE 모뎀에 직접 유선(LAN)으로 연결**해도 동일한 효과를 얻을 수 있다. <br>Wi-Fi 는 무선 접근의 편의를 위한 옵션일 뿐이다.

SSID 와 비밀번호는 편한 대로 작성하여 사용하면 된다.

---

## 설정 완료 후 점검 항목

LTE 모뎀의 모든 설정이 완료되면, **외부 네트워크에서 DDNS 도메인을 통해 EdgePC 의 소프트웨어에 접근할 수 있는지** 확인한다.

아래 URL 중 **하나 이상이 정상 응답**하면 LTE 모뎀의 외부 접속(DDNS + DMZ) 이 정상 동작하는 것으로 간주한다.  
 `{host}` 는 [3. DDNS 설정](#3-ddns-설정) 에서 결정한 호스트명(예: `tapfreeE0801.cns-link.net`) 으로 치환한다.

| 항목 | 구동 확인 URL (http, https 유의) | 성남 캠퍼스 예시 | 비고 |
|---|---|---|---|
| **모뎀 설정 페이지** | `http://{host}:8081/{path}` | `http://tapfreeE0801.cns-link.net:8081` | [모뎀 설정 페이지 외부 포트(8081)와 Path](#5-모뎀-설정-페이지-외부-port-설정) 사용 |
| **Geospace** | `https://{host}:8443/` | `https://tapfreeE0801.cns-link.net:8443/` | |
| **PRM** | `http://{host}:7081/prm/login` | `http://tapfreeE0801.cns-link.net:7081/prm/login` | |
| **Tapfree-edge** | `https://{host}:9443/` | `https://tapfreeE0801.cns-link.net:9443/` | |
| **Quuppa** | `http://{host}:8080/qpe/` | `http://tapfreeE0801.cns-link.net:8080/qpe/` | |

> 점검은 **외부망에 연결된 단말**(LTE 회선의 스마트폰, 외부망 노트북 등) 에서 수행한다.
>
> 어느 URL 도 응답하지 않으면 [3. DDNS 설정](#3-ddns-설정) 또는 [4. DMZ 설정](#4-dmz-설정) 을 재점검한다.

### SSH 접속 확인

DDNS 도메인을 통해 EdgePC 에 **SSH 로도 접속 가능한지** 확인한다. 위 표의 HTTP/HTTPS 점검과 동일한 외부망 단말에서 수행.

- `root` 계정 (비밀번호 `wldhvmffos#123`) 으로 접속하여 로그인되면 정상.
- 연결 거부(Connection refused) 또는 타임아웃 시 [4. DMZ 설정](#4-dmz-설정) 의 대상 IP 가 EdgePC(`192.168.0.30`) 로 되어 있는지 재확인.
